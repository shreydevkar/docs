---
title: "GridOS error reference: keys, guards, and load failures"
description: "Solutions for the most frequent GridOS errors: missing API keys, rejected keys, provider overload, formula guard blocks, missing save files, and blank charts."
---

Most GridOS errors fall into a small set of categories — provider configuration, formula guard blocks, and save/load issues. Use the reference below to identify the error you are seeing and apply the fix.

<AccordionGroup>
  <Accordion title='"No LLM provider is configured" (400)'>
    **What it means:** GridOS has no API key for any provider. The AI agent cannot run without at least one configured key.

    **How to fix it:**

    1. Click the gear icon in the GridOS menubar to open Settings.
    2. Paste an API key for at least one provider (Groq is free and requires no credit card).
    3. Save and close Settings.
    4. The model picker in the chat composer will now show available models. Send your prompt again.

    Alternatively, create a `.env` file in the repository root with at least one of the following before starting the server:

    ```bash
    GOOGLE_API_KEY=your_key
    ANTHROPIC_API_KEY=your_key
    GROQ_API_KEY=your_key
    OPENROUTER_API_KEY=your_key
    ```
  </Accordion>

  <Accordion title='"The API key for this model was rejected" (401)'>
    **What it means:** The provider returned a `401 Unauthorized` response. The key you configured is invalid, expired, or belongs to a different provider.

    **How to fix it:**

    1. Click the gear icon to open Settings.
    2. Find the provider whose key was rejected.
    3. Paste a fresh, valid key and save.
    4. Retry your prompt.

    A bad key for one provider does not affect other providers. If your Groq key is invalid but your Gemini key is valid, Gemini models continue working while you fix the Groq key.
  </Accordion>

  <Accordion title='"Model provider is temporarily overloaded" (503)'>
    **What it means:** The provider returned a transient error (rate limit, server overload, or gateway timeout). GridOS already retried the request four times with exponential backoff (~1s, ~2s, ~4s intervals) before surfacing this error.

    **How to fix it:**

    Wait a moment, then send your prompt again. Transient overloads on Groq and OpenRouter free tiers are common during peak hours. If the error persists, switch to a different provider in the model picker — for example, switch from Groq to Gemini while Groq recovers.
  </Accordion>

  <Accordion title='Formula guard: "agent proposed formulas whose inputs are empty" (422)'>
    **What it means:** The AI proposed one or more formulas that reference cells which are currently empty. Applying them would produce `#DIV/0!` errors or misleading zeros. GridOS blocks the write before it touches the sheet.

    The error message lists the affected cells and their empty references, for example:

    ```
    C4 (=DIVIDE(C3, B3)) references empty cell(s): C3, B3
    ```

    **How to fix it:** Choose one of the following approaches:

    - **Re-ask the agent:** Tell the agent to also populate the referenced cells in the same response. For example: *"Fill in C3 and B3 with the baseline values first, then add the formula in C4."*
    - **Fill the cells manually:** Type values into the empty referenced cells yourself, then re-send your original prompt.
  </Accordion>

  <Accordion title='"No save file found" on load'>
    **What it means:** You clicked Load (or the server tried to auto-load on startup) but no `system_state.gridos` file exists yet. This happens on a fresh install or after the save file has been deleted.

    **How to fix it:** This is not an error — it simply means no previous session was saved. Start fresh by describing what you want to build in the chat, or open one of the preset templates from the template library. Save your workbook after making changes using **File → Save** to create a save file for future loads.
  </Accordion>

  <Accordion title="Chart not appearing after the agent adds one">
    **What it means:** The agent wrote a `chart_spec` but the chart did not render in the workbook. The most common cause is that the data range referenced in the chart spec contains no data — the chart renderer has nothing to plot.

    **How to fix it:**

    1. Check that the cells in the data range actually contain values. Open the range (e.g. `A1:B6`) in the grid and confirm the cells are non-empty.
    2. If the cells are empty, ask the agent to populate the data first, then ask it to add the chart: *"Fill A1:B6 with the revenue data, then add a bar chart."*
    3. If the data is present but the chart still does not appear, try asking the agent to re-create the chart by referring to it by name — GridOS upserts charts by title, so re-issuing the same chart spec updates the existing chart in place.
  </Accordion>
</AccordionGroup>
